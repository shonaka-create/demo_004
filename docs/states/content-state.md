# 状態遷移図 — コンテンツ管理（INSIGHT / CASE）

## INSIGHT 記事の状態遷移

```mermaid
stateDiagram-v2
    [*] --> 下書き : 新規投稿作成

    下書き --> レビュー待ち : 担当者が「レビュー依頼」送信
    下書き --> 公開済み : 管理者が直接公開

    レビュー待ち --> 修正中 : レビュアーが差し戻し
    レビュー待ち --> 公開済み : レビュアーが承認・公開

    修正中 --> レビュー待ち : 再提出
    修正中 --> 下書き : 保存のみ

    公開済み --> 非公開 : 管理者が非公開に変更
    公開済み --> 更新中 : 内容を編集（保存前）
    更新中 --> 公開済み : 更新を保存
    更新中 --> 下書き : 変更を破棄して下書きに戻す

    非公開 --> 公開済み : 再公開
    非公開 --> ゴミ箱 : 削除操作

    ゴミ箱 --> [*] : 完全削除（30日後自動）
    ゴミ箱 --> 非公開 : 復元

    note right of 公開済み
        フロントエンドに表示
        一覧ページ・記事詳細に反映
        TOPページのINSIGHT PICKUPにも表示（最新3件）
    end note
```

## CASE 事例の状態遷移

```mermaid
stateDiagram-v2
    [*] --> 下書き : 新規事例作成（ACFフィールド入力）

    下書き --> 確認待ち : クライアント確認依頼
    確認待ち --> 修正中 : クライアントから修正依頼
    確認待ち --> 公開承認済み : クライアント承認

    修正中 --> 確認待ち : 再確認依頼
    修正中 --> 下書き : 保存のみ

    公開承認済み --> 公開済み : 管理者が公開操作

    公開済み --> 非公開 : 事例の取り下げ要請
    公開済み --> 更新中 : 成果数値等の更新
    更新中 --> 公開済み : 更新保存

    非公開 --> ゴミ箱 : 完全削除操作
    ゴミ箱 --> [*] : 完全削除

    note right of 確認待ち
        クライアント（掲載対象法人）の
        事前確認が必要
        確認なし公開はリスクあり
    end note

    note right of 公開済み
        CASEアーカイブページに表示
        業種フィルターで分類
        TOPページのCASE TEASERにも表示
    end note
```

## WordPress コンテンツ状態マッピング

| 上記状態 | WordPress ステータス | 備考 |
|---------|-------------------|------|
| 下書き | `draft` | 管理画面のみ表示 |
| レビュー待ち / 確認待ち | `pending` | Pending Review |
| 公開済み | `publish` | フロントに表示 |
| 非公開 | `private` | 管理者のみ閲覧可 |
| ゴミ箱 | `trash` | 30日後自動削除 |
| 修正中 / 更新中 | `draft`（新リビジョン） | WordPressリビジョン管理 |

## CASE カスタムフィールド（ACF）の入力完了状態

CASE は公開前に以下フィールドが全て入力済みであること：

| フィールド | 必須 |
|-----------|------|
| 業種 (`case_industry`) | ✓ |
| 施設規模 (`case_scale`) | — |
| 地域 (`case_region`) | — |
| 課題（支援前） (`case_challenge`) | ✓ |
| 成果（数値） (`case_result_number`) | ✓ |
| 成果（説明） (`case_result_text`) | ✓ |
| クライアントの声 (`case_testimonial`) | — |
| アイキャッチ画像 | ✓ |
