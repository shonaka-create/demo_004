# 批判的レビュー v1 — フェンリルイノベーションズ コーポレートサイト

> **実施日**: 2025年  
> **対象**: PRD v1 + デザイン設計書  
> **方法**: PRD・設計書を別視点から批判的に検証

---

## 1. 市場・ビジネス面

### 1.1 指摘事項

**🔴 高リスク**

| 指摘 | 詳細 | 対策 |
|------|------|------|
| 会社情報が未確定 | 代表者名・所在地・資本金が全て TBD。登記前にサイト公開するリスクがある | 登記完了まで「準備中」プレースホルダーを維持。公開日を登記完了後に設定 |
| 実績ゼロのスタートアップ | CASE（事例）・数値実績が初期はゼロ。「変革の現場から」セクションがプレースホルダーのまま公開される可能性 | ローンチ前に最低1件の事例または代替コンテンツ（代表のコラム等）を準備 |

**🟡 中リスク**

| 指摘 | 詳細 | 対策 |
|------|------|------|
| ターゲットが「経営幹部（40〜60代）」なのにモダンUI | ターゲット層がダーク系 Gold アクセントの高級感を好むか検証が必要 | MVP後にユーザーテストを実施し、ファーストインプレッションを確認 |
| 競合との差別化が不明確 | 「社会システムの設計者」という言葉は抽象的。競合コンサル会社との違いを具体的に示す必要がある | SYSTEM DESIGN ページに差別化比較表を必ず実装（PRD 4.2 に記載あり） |

## 2. 機能・設計面

### 2.1 指摘事項

**🔴 高リスク**

| 指摘 | 詳細 | 対策 |
|------|------|------|
| 問い合わせ後の対応フローが未定義 | サンクスメッセージの次のステップ（いつ返信？担当者は誰？）がサイト上で示されていない | CONTACT ページに「3営業日以内にご返信します」等を明記 |
| reCAPTCHA 未設定 | スパムボット対策なしで Contact Form 7 を公開するのは危険 | reCAPTCHA v3 を CF7 設定時に必ず導入（PRD 10. リスク対策に追記済み） |

**🟡 中リスク**

| 指摘 | 詳細 | 対策 |
|------|------|------|
| INSIGHT 初期コンテンツが不明 | ローンチ時に記事ゼロだとINSIGHTページが空欄になる | 最低3〜5本の記事をローンチ前に作成（テーマは業界課題・DX・事例解説） |
| BUSINESS 詳細4ページのコンテンツ | 詳細ページのサービス内容・支援の流れが現在素材のみで文章未確定 | 各詳細ページのコピーライティングを先行して確定させる |
| 採用情報の職種カード | 募集ポジション・給与レンジが未確定のまま実装している | `【準備中】` タグで仮表示し、確定次第更新 |

**🟢 低リスク（改善推奨）**

| 指摘 | 改善案 |
|------|--------|
| ハンバーガーメニューのアクセシビリティ | `aria-expanded`, `aria-label` を必ず実装。スクリーンリーダー対応 |
| フォームの `autocomplete` 属性 | `name`, `email`, `tel` フィールドに適切な `autocomplete` 値を設定 |
| INSIGHT カード画像の `alt` テキスト | WordPress 投稿時にメディアライブラリで `alt` を設定する運用ルールを定める |

## 3. 技術・実現性面

### 3.1 指摘事項

**🔴 高リスク**

| 指摘 | 詳細 | 対策 |
|------|------|------|
| Emanon PREMIUM との CSS 競合 | カスタム CSS が Emanon のデフォルトスタイルと `specificity` で競合する可能性大 | 子テーマ内でのみカスタマイズ・`!important` の使用を最小化・詳細セレクタを優先 |
| WP Mail SMTP 未設定リスク | さくらサーバーの PHP `mail()` 関数はスパム判定されやすい。CF7 メールが届かない事例多数 | WP Mail SMTP または Fluent SMTP を導入し、Googleワークスペース or SendGrid 経由でメール送信 |

**🟡 中リスク**

| 指摘 | 詳細 | 対策 |
|------|------|------|
| さくら共有サーバーのPHP版本 | PHP 7.4以下ではプラグイン互換問題が発生 | コントロールパネルで PHP 8.1 以上を確認してから WordPress インストール |
| 画像最適化未設定 | PageSpeed 80 以上を目指す場合、WebP変換・遅延読み込みが必須 | WebP Express + Lazy Load を WordPress プラグインで設定 |
| カスタム投稿タイプ `case` の予約語リスク | WordPress に `case` という予約語はないが、スラッグとの競合に注意 | CPT UI で `case` スラッグを確認し、アーカイブ URL が `/case/` になることを検証 |

## 4. レビュー後アクション

### 要対応（ローンチ前必須）

- [ ] 登記完了まで公開しない（会社情報確定が条件）
- [ ] reCAPTCHA v3 を Contact Form 7 に必ず設定
- [ ] WP Mail SMTP を設定し、メール送信テストを実施
- [ ] INSIGHT 初期記事を最低3本用意
- [ ] CONTACT ページに対応日時の目安を明記
- [ ] PHP 8.1以上を確認してから WordPress インストール

### 推奨改善（ローンチ後対応可）

- [ ] ハンバーガーメニューのアリア属性追加
- [ ] フォームフィールドの `autocomplete` 属性追加
- [ ] 代表者写真・会社写真の素材収集
- [ ] Google Analytics 設置
- [ ] ユーザビリティテスト実施（ターゲット層への確認）
